## 第一部分：项目概述与核心定位

### 1.1 项目本质定位

**长篇小说AI助手平台**不是通用AI写作工具，而是**网文创作者专属的AI创作提效中台**。

**核心价值主张**：
- 解决长篇网文创作的四大核心痛点：设定管理混乱、长篇崩设定、创作效率低下、工具分散
- 通过"12大弹性要素体系"和"双层知识库系统"，让AI真正懂作者的作品
- 提供从设定搭建→情节铺陈→章节生成→内容优化的全流程提效

**平台本质属性**：
- 纯工具定位，无社交功能
- 数据完全私有，用户可控
- 聚焦长篇网文创作（百万字级）
- 个人离线式创作辅助工具

### 1.2 核心功能边界

#### 必须做的核心功能（P0级）：
1. **设定统一管理系统**：12大弹性要素，修改一处自动同步
2. **双层知识库系统**：个人专属知识库 + 单本小说知识库
3. **基础章节生成**：基于设定自动生成章节内容
4. **数据安全隔离**：用户数据完全隔离，无社交功能

#### 可以做的扩展功能（P1-P3级）：
1. **智能提醒与矛盾检测**：设定一致性检查
2. **批量生成功能**：任务队列，进度跟踪
3. **可视化图谱**：人物关系、势力层级、感情线
4. **导出备份功能**：多格式导出，自动备份

#### 坚决不做的功能：
1. **社交功能**：用户关注、私信、评论、分享
2. **社区功能**：论坛、聊天室、讨论区、排行榜
3. **用户互动**：点赞、收藏、打赏、互动
4. **过度复杂功能**：实时协作编辑、高级AI训练、多模态生成

### 1.3 三层弹性要素体系

#### 第一层：基础核心要素（6个，所有题材通用）
1. **核心人设**：主角、主要配角的核心设定
2. **世界观基底**：世界背景、基本规则、社会结构
3. **核心主线**：故事的主要发展脉络
4. **核心冲突**：主要矛盾和冲突点
5. **关键道具/信物**：重要物品及其关联
6. **核心场景**：关键场景和地点

#### 第二层：题材扩展要素（按题材分类）
- **玄幻/仙侠**：势力体系、修炼等级、丹药/法宝体系
- **现代言情**：感情线、人物羁绊、家庭背景
- **悬疑/推理**：伏笔线索、案件脉络、嫌疑人设定
- **都市爽文**：金手指规则、逆袭节点、反派阵营
- **科幻/星际**：科技体系、星际势力、种族设定
- **历史穿越**：时代背景、历史人物、朝代规则

#### 第三层：自定义要素（作者按需创建，无数量限制）
- 作者可以自定义任何需要的要素类型
- 支持自由添加、编辑、删除
- 与前两层要素完全联动

### 1.4 双层知识库系统

#### 个人专属知识库
- 存储作者固定创作风格
- 存储常用设定模板
- 存储个人快捷指令
- 跨作品复用，提升效率

#### 单本小说知识库
- 存储该小说的12大弹性要素
- 存储大纲/细纲
- 存储已生成章节
- 与设定实时同步

#### 联动逻辑
- AI生成时同时调取两层知识库
- 个人知识库提供风格和常用设定
- 小说知识库提供情节和具体设定
- 确保生成内容既符合风格又贴合情节

### 1.5 核心创作流程

#### 阶段1：新建小说
1. 选择题材
2. 加载对应模板
3. 填写基本信息
4. 创建初始设定

#### 阶段2：框架搭建
1. 填充核心要素
2. 编辑扩展要素
3. 创建自定义要素
4. 生成可视化图谱

#### 阶段3：大纲设计
1. AI生成总大纲
2. 分卷大纲设计
3. 章节细纲规划
4. 逻辑关系校验

#### 阶段4：章节生成
1. 批量生成章节
2. 人工轻量介入
3. 智能润色优化
4. 合规校验检查

#### 阶段5：导出发布
1. 多格式导出
2. 平台适配转换
3. 自动备份存档
4. 版本管理记录

### 1.7 项目核心原则

4. **数据安全优先**：用户数据完全私有、安全、可控

1. **测试驱动**：先写测试，再写代码
2. **持续集成**：每日构建，及时发现错误
3. **代码审查**：所有代码必须经过审查

### 2.2 技术栈详细选型

#### 前端技术栈
- **框架**：React 18 + Next.js 14
  - 选择理由：成熟生态、SSR支持、性能优秀
  - 版本要求：React 18.2+、Next.js 14.0+
  
- **语言**：TypeScript
  - 选择理由：类型安全、开发体验好、生态完善
  - 版本要求：TypeScript 5.0+
  
- **样式**：Tailwind CSS + shadcn/ui
  - 选择理由：原子化CSS、组件化开发、响应式支持
  - 版本要求：Tailwind 3.3+、shadcn/ui 0.4+
  
- **可视化**：D3.js / React Flow
  - 选择理由：成熟稳定、功能强大、社区活跃
  - 版本要求：D3.js 7.8+、React Flow 11.7+
  
- **状态管理**：Zustand
  - 选择理由：轻量简洁、性能优秀、API友好
  - 版本要求：Zustand 4.4+
  
- **构建工具**：Vite
  - 选择理由：极速启动、按需编译、开发体验好
  - 版本要求：Vite 5.0+

#### 后端技术栈
- **框架**：Node.js + NestJS + TypeScript
  - 选择理由：JavaScript全栈、架构清晰、模块化好
  - 版本要求：Node.js 18+、NestJS 10.0+
  
- **数据库**：PostgreSQL 15 + Redis 7
  - 选择理由：关系型+缓存组合、性能优秀、功能强大
  - 版本要求：PostgreSQL 15.0+、Redis 7.0+
  
- **ORM**：Prisma
  - 选择理由：类型安全、查询优化、迁移管理
  - 版本要求：Prisma 5.0+
  
- **认证**：JWT + bcrypt
  - 选择理由：轻量安全、成熟稳定、易于集成
  - 版本要求：jsonwebtoken 9.0+、bcrypt 5.1+
  
- **任务队列**：BullMQ
  - 选择理由：Redis驱动、功能完善、性能优秀
  - 版本要求：BullMQ 5.0+
  
- **HTTP客户端**：Axios
  - 选择理由：成熟稳定、功能完整、易于使用
  - 版本要求：Axios 1.6+

#### AI集成层
- **豆包 API**（字节跳动）
  - 适用场景：大纲/细纲生成、人物设定/感情线设计
  - 配置要求：API_KEY、API_URL、MODEL
  
- **文心一言**（百度）
  - 适用场景：世界观/势力体系生成、专业设定、去AI味润色
  - 配置要求：API_KEY、API_URL、MODEL
  
- **通义千问**（阿里）
  - 适用场景：章节正文批量生成、情节续写、批量润色
  - 配置要求：API_KEY、API_URL、MODEL
  
- **智谱清言**（智谱AI）
  - 适用场景：悬疑/推理题材情节设计、细节补充
  - 配置要求：API_KEY、API_URL、MODEL

#### 基础设施
- **容器化**：Docker + Docker Compose
  - 选择理由：环境一致性、部署简单、易于扩展
  - 版本要求：Docker 24.0+、Docker Compose 2.20+
  
- **部署环境**：阿里云 ECS + OSS
  - ECS配置：8核16G（起步），可弹性扩展
  - OSS配置：标准存储，自动备份
  
- **监控**：Prometheus + Grafana
  - 选择理由：开源成熟、监控告警完善
  - 版本要求：Prometheus 2.45+、Grafana 10.0+
  
- **日志**：ELK Stack
  - 选择理由：日志收集、分析、可视化一站式
  - 版本要求：Elasticsearch 8.8+、Logstash 8.8+、Kibana 8.8+
  
- **CI/CD**：GitHub Actions
  - 选择理由：集成度高、配置简单、免费额度充足

#### 小说API
```typescript
// 创建小说
POST /api/v1/novels
{
  "title": "小说标题",
  "category": "玄幻",
  "pen_name": "笔名"
}

// 获取小说列表
GET /api/v1/novels?page=1&pageSize=20&status=draft

// 获取小说详情
GET /api/v1/novels/:id

// 更新小说
PUT /api/v1/novels/:id
{
  "title": "新标题",
  "category": "仙侠"
}

// 删除小说
DELETE /api/v1/novels/:id

// 更新小说状态
PUT /api/v1/novels/:id/status
{
  "status": "writing" | "completed"
}

// 获取小说统计
GET /api/v1/novels/:id/stats
```
