## 📋 项目概述

本项目旨在为网文创作者打造一个**纯工具型**的AI创作提效平台，解决长篇网文创作中的四大核心痛点：

1. **设定管理混乱** - 人物、世界观、道具等设定分散，修改一处需手动更新多处
2. **长篇崩设定** - 百万字长篇创作中，设定前后不一致问题频发
3. **创作效率低下** - AI生成内容质量不高，需要大量人工修改
4. **工具分散** - 缺乏一体化的创作辅助工具

## 🎯 核心功能模块

### 1. 12大弹性要素体系
- **基础核心要素**（6个）：核心人设、世界观基底、核心主线、核心冲突、关键道具/信物、核心场景
- **题材扩展要素**（按题材分类）：玄幻/仙侠、现代言情、悬疑/推理、都市爽文、科幻/星际、历史穿越
- **自定义要素**（无数量限制）：作者按需创建

### 2. 双层知识库系统
- **个人专属知识库**：存储作者固定创作风格、常用设定模板、个人快捷指令
- **单本小说知识库**：存储该小说的12大弹性要素、大纲/细纲、已生成章节

### 3. 核心功能模块
- **设定管理系统**：12大弹性要素，修改一处自动同步
- **知识库管理系统**：双层知识库，AI生成时自动调用
- **章节生成系统**：单章生成、批量生成、智能润色、情节续写
- **可视化图谱系统**：人物关系图谱、势力层级图、感情线时间轴、情节发展图
- **任务队列系统**：异步处理AI生成任务，进度实时跟踪
- **智能提醒系统**：设定矛盾检测、情节逻辑检查、创作进度提醒、智能建议
- **数据导出系统**：多格式导出（Markdown/Word/PDF/JSON）、平台适配、自动备份

## 🛠️ 技术栈

### 前端技术栈
- **框架**: React 18 + Next.js 14
- **语言**: TypeScript 5.0+
- **样式**: Tailwind CSS + shadcn/ui
- **可视化**: D3.js / React Flow
- **状态管理**: Zustand
- **构建工具**: Vite

### 后端技术栈
- **框架**: Node.js + NestJS + TypeScript
- **数据库**: PostgreSQL 15 + Redis 7
- **ORM**: Prisma 5.0+
- **认证**: JWT + bcrypt
- **任务队列**: BullMQ
- **HTTP客户端**: Axios

### AI集成层
- **豆包 API**（字节跳动）：大纲/细纲生成、人物设定/感情线设计
- **文心一言**（百度）：世界观/势力体系生成、专业设定、去AI味润色
- **通义千问**（阿里）：章节正文批量生成、情节续写、批量润色
- **智谱清言**（智谱AI）：悬疑/推理题材情节设计、细节补充

### 基础设施
- **容器化**: Docker + Docker Compose
- **部署环境**: 阿里云 ECS + OSS
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack
- **CI/CD**: GitHub Actions

## 📊 项目核心原则

### 技术原则
1. **成熟优先**：选择经过生产验证的技术
2. **简单优先**：能用简单方案解决的不复杂化
3. **可维护优先**：代码可读、可测试、可扩展
4. **性能优先**：响应速度、并发能力、资源消耗

### 产品原则
1. **用户痛点优先**：解决真实痛点，不是创造伪需求
2. **专注核心优先**：聚焦核心功能，不盲目扩展
3. **体验优先**：操作简单、界面简洁、流程顺畅
4. **数据安全优先**：用户数据完全私有、安全、可控

### 开发原则
1. **测试驱动**：先写测试，再写代码
2. **持续集成**：每日构建，及时发现错误
3. **代码审查**：所有代码必须经过审查
4. **文档同步**：代码和文档同步更新
